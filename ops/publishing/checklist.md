# 发布前检查清单

## 内容审查

- [ ] 标题是否符合公式：[数字/痛点/名人] + [方案] + [开源/首发]
- [ ] 开头是否有痛点场景制造代入感
- [ ] 架构图是否清晰（一张图讲清全局）
- [ ] 是否有竞品对比（诚实、不吹不黑）
- [ ] 是否有快速开始命令（可复制、一步跑通）
- [ ] 是否坦诚不足（建立信任）
- [ ] 是否有 CTA（Star / Issue / 讨论）

## 技术审查

- [ ] 所有命令可运行（git clone / pip install / python 脚本）
- [ ] 所有链接有效（GitHub / 文档 / 参考）
- [ ] 代码块语言标注正确
- [ ] 无敏感信息暴露（API key / 私人路径 / 内部数据）

## 配图

- [ ] 封面 800×420（掘金规格）
- [ ] 架构图清晰可读
- [ ] 流程图逻辑正确
- [ ] 所有图片已上传到平台图床

## 平台适配

- [ ] 掘金版：标题 ≤ 50 字、标签 3-5 个、摘要 100 字内
- [ ] 知乎版：开头用"谢邀"或直接切入、去 H1 标题、加话题标签
- [ ] 两版本内容一致、格式差异已处理

## 发布后

- [ ] 确认 GitHub README 有文章链接
- [ ] 确认 CI badges 正常显示
- [ ] 24 小时内回复所有评论
