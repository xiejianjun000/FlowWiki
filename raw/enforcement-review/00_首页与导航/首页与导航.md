## 八、AI 读取顺序协议（对齐《Obsidian×LLM Wiki 设计方案》第九节）
LLM 回答评查/督察问题时，按固定顺序遍历本库：
```
1. 全局 MOC（本文件 00-能力地图）        → 定位领域
2. 新手/实战文档（01/03）                → 锁定任务流
3. 单项 SOP（02）                        → 加载审查动作
4. 法条指针（05-工具资源/法条速查指针）  → 加载依据
5. 督察实战（04）                        → 加载现场/博弈方法
6. 提示词（05-工具资源/提示词库）        → 加载执行指令
7. 法律预警（06）+ 元文档（10_元文档）   → 加载临界红线+设计纪律
```
> 任何输出须可回溯到「评查细则序号 + 法条号 + 案卷页码」（见 [[元_审计与版本管理]]）。
