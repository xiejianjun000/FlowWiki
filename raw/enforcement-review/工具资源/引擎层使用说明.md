## 何时读我（触发信号）
- 拿到一份案卷/督察任务，不确定该"直接说需求"还是"复制提示词"
- 听说库里有 skill 又有提示词，搞不清谁负责什么
- 想给同事/团队讲清楚这套知识库怎么用

## 一句话定位
**知识库文档 = 方法（该怎么做）；Skill = 自动门（系统推你进门）；提示词 = 遥控器（你主动按）。**
三者是同一套知识库的"方法层 + 两种调用入口"，最终都读同一批文档、产出同一类成果。

## 二、Skill 是什么 · 何时用
**本质**：写在 `~/.qclaw/skills/` 下的路由程序。系统在会话中识别到你消息里的触发词（如"案卷评查""督察""法典8-15"），**自动读取 SKILL.md**，我据此先读 MOC、再定位文档、按步骤执行。

**本库涉及 4 个 skill**：
- `eco-review-kb` —— 本库总路由（先读 00-能力地图 → 分发到对应文档）
- `eco-enforcement` —— 执法检查技术细节
- `eco-enforcement-platform` —— 一体化入口（检查/文书/评查/企业）
- `hunan-eco-penalty-docs` —— 湖南省处罚文书范本

**触发时机**：你在对话里**直接说需求**，无需任何操作。例如：
> "帮我评查这份危废案卷" → 命中 eco-review-kb → 我自动读 MOC → 调 `03-实战全流程/移送涉嫌犯罪案卷评查.md` → 逐步执行

**特点**：零操作、隐式、适合日常随口提需求；你甚至不需要知道文件在哪。

---

## 三、提示词是什么 · 何时用
**本质**：把一篇文档的方法，压缩成一句**可复制发送的指令**（四段式：场景/要求/材料/输出），`<变量>` 处填实际案卷材料即可。

**本库提示词分布**：
- 集中索引：`05-工具资源/提示词库/提示词索引.md`（P1-P10，评查/督察/文书三类）
- 分散内嵌：每篇文档结尾的「▶️一键提示词」

**触发时机：你主动复制 → 替换变量 → 发送**。适合以下场景：
1. **精确控制输入**：只想"扫红线不打分"，或只跑"9种信号识别"单动作
2. **给团队分发**：把 P1 发给同事，口径一致、不依赖他懂知识库
3. **批量处理**：同类案卷逐个套同一提示词
4. **脱离结构单跑**：不想走整套路由，直接要某个产出

**特点**：显式、可控、可分发；你握有输入变量的完全控制权。

## 四、二者关系（重点）
- **同一知识库的两种入口**，不是两套东西。Skill 是"自动门"（系统推），提示词是"遥控器"（你按）。
- 最终都 read 同一批 `00-06` 文档，产出同一类 ⚖️/🔍/📊 成果。
- **分工差异**：
  - Skill 偏"**路由 + 方法引导**"——帮你找对文档、走对流程
  - 提示词偏"**临门一脚的具体执行指令**"——把方法落成可发的一句话

## 五、协同干活闭环（实际怎么用）
```
① 你随口说需求 ──→ Skill 自动路由到文档（你不用找文件）
        │
② 文档给出：目标拆解 + 资料清单 + 全流程步骤 + 常见扣分点
        │
③ 你复制文档内「▶️一键提示词」或 提示词索引对应条 → 填入案卷材料
        │
④ 发送 ──→ 我按四段结构执行 ──→ 输出 ⚖️ 合规守门报告
        │
⑤ （可选）若你不要精确控制，直接说需求让 Skill 跑完也行
```

## 八、三类调用模式（澄清混淆）
实际干活只有三种组合，绝大多数情况走第二种：

1. **纯 Skill 模式**（随口说，系统自动路由）
   - 场景：日常评查/查询/法律预警咨询，你不想找文件。
   - 例：「帮我评查这份危废案卷」→ `eco-review-kb` 自动读 MOC → 调 `移送涉嫌犯罪案卷评查.md` → 你给材料我执行。
   - 特点：零操作、隐式；但输入变量由你口头给，精度靠对话。

2. **Skill 路由 + 提示词精确执行**（⭐ 绝大多数推荐）
   - 场景：既要自动找对文档，又要精确控制输入/输出、可复现、可分发。
   - 流程：Skill 帮你定位文档 → 你复制该任务对应提示词（见上表 P 列）→ 填 `< >` 变量发回 → 我按四段结构执行。
   - 特点：路由准 + 执行可控，是「方法层 + 两种入口」协同闭环的标准用法。

3. **纯提示词模式**（不走路由，直接发指令）
   - 场景：批量处理同类案卷、只跑单动作（如仅扫红线）、给团队分发标准化指令、脱离结构单跑。
   - 例：把 P1 发给同事，他不必懂知识库也能口径一致地扫红线。
   - 特点：显式、可控、可分发；但需你自己知道套哪条提示词。

> 决策口诀：**日常随手 → 纯 Skill；要准要复现 → Skill+提示词；批量/分发/单动作 → 纯提示词。**

## 九、常见误区
- ❌ "有 skill 就不用提示词" → 错。skill 管路由，提示词管精确执行；要控制输入就得用提示词。
- ❌ "提示词能替代知识库" → 错。提示词是文档方法的压缩，文档才是权威依据，提示词里 `→ 配套文档` 都指回库。
- ❌ "skill 会自动把案卷评完" → 半对。skill 路由+引导，但案卷材料得你给（粘贴/附件），缺材料我不臆测。
