# 定量分析 vs 定性分析

## 对比维度

| 维度 | 定量分析 | 定性分析 |
|------|---------|---------|
| 数据类型 | 数值型数据 | 文本、行为、情境 |
| 方法 | 统计、回归、机器学习 | 访谈、观察、案例分析 |
| 输出 | 数值结论、置信区间 | 描述性结论、模式识别 |
| 优势 | 客观、可重复、可量化 | 深入理解、发现隐含因素 |
| 劣势 | 可能遗漏非量化因素 | 主观性强、难以泛化 |

## 结论

根因分析中通常先做定量分析缩小范围，再用定性分析深入理解原因。

## 深入对比

### 定量分析的核心技术

1. **描述性统计**：均值、方差、分位数、分布形态 → 快速了解数据全貌
2. **假设检验**：t 检验、卡方检验、ANOVA → 验证差异是否统计显著
3. **回归分析**：线性回归、逻辑回归、多元回归 → 量化自变量对因变量的影响
4. **时间序列分析**：趋势、周期、异常点检测 → 发现时序规律
5. **实验设计**：A/B 测试、因子设计 → 严格验证因果关系

### 定性分析的核心技术

1. **事件访谈**：与相关人员沟通获取操作细节和上下文
2. **时间线复盘**：按时间顺序梳理事件经过，还原决策逻辑
3. **流程审查**：对照标准流程检查实际操作中的偏差
4. **文档分析**：审阅操作手册、变更工单、会议记录
5. **情境模拟**：还原现场环境，观察操作者的行为模式

### 混合分析工作流

| 阶段 | 方法 | 行动 |
|------|------|------|
| 1. 问题识别 | 定性 | 从用户反馈/告警中理解问题背景 |
| 2. 范围界定 | 定量 | 通过数据分布和占比分析确定影响面 |
| 3. 假设生成 | 定性 | 结合领域知识和异常模式提出假设 |
| 4. 假设筛选 | 定量 | 用统计数据排除不成立的假设 |
| 5. 深入验证 | 定性 | 对剩余假设进行人工核查和逻辑推理 |
| 6. 结论输出 | 定量+定性 | 数据证据 + 业务解释一体呈现 |

### 定量分析的局限性

1. **数据质量依赖**：输入数据有偏 → 结论有偏（Garbage In, Garbage Out）
2. **相关性陷阱**：统计显著 ≠ 因果成立，需警惕混淆变量
3. **过度简化**：复杂的业务逻辑可能被数学模型过度抽象化
4. **幸存者偏差**：只能分析被记录的数据，无法涵盖未记录的隐式信息

### 定性分析的局限性

1. **主观偏差**：分析者的经验和偏好影响判断
2. **样本有限**：访谈和案例分析能覆盖的样本量远小于定量分析
3. **可复现性差**：不同分析者面对相同材料可能得出不同结论
4. **效率较低**：人工分析和推理速度远慢于自动化定量计算

## 选择策略

```
问题的数据化程度如何？
├─ 高度数据化（有大量结构化数据）→ 定量为主，定性为辅
├─ 中度数据化（有部分数据但不完整）→ 定量定位 + 定性解释
└─ 低度数据化（主要靠经验和描述）→ 定性为主，定量作为参考
```
