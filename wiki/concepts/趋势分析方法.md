# 趋势分析方法

## 定义

趋势分析方法是通过时间序列数据的统计分析，识别数据变化的方向、速率和周期性模式，为异常检测和预测提供依据的方法体系。

## 主要方法

- **移动平均**：平滑短期波动，揭示长期趋势
- **线性回归**：量化变化速率
- **季节性分解**：分离趋势、周期和残差

## 相关资料

- **ARIMA/SARIMA**：自回归积分滑动平均模型，适合平稳时间序列的短期预测，能同时建模趋势和季节性
- **指数平滑 (ETS)**：通过加权历史数据，近期数据权重更高，适用于无明显趋势和周期的序列
- **Prophet**：Facebook 开源的时序预测工具，自动处理节假日效应、趋势变化点和缺失值，适合业务分析
- **STL 分解**：基于 LOESS 的季节性趋势分解，比经典分解更鲁棒，能处理变周期的季节性模式
- **断点检测**：使用 PELT 或 Binary Segmentation 算法识别时间序列的结构性变化点，辅助定位异常发生的精确时点

## 工具生态

| 工具 | 用途 | 特点 |
|------|------|------|
| Python statsmodels | ARIMA、STL、VAR 建模 | 统计推断能力强，结果可解释 |
| Facebook Prophet | 业务时序预测 | 自动化程度高，内置节假日处理 |
| R forecast 包 | ETS、ARIMA、TBATS | 学术验证充分，适合研究场景 |
| Grafana + Prometheus | 系统监控趋势 | 实时可视化，告警规则联动 |
| Tableau / Power BI | 业务报表趋势线 | 低代码，业务人员可直接使用 |

## 趋势分析的典型陷阱

1. **伪相关陷阱**：两个独立的时间序列可能因共同的时间趋势呈现高度相关（如冰淇淋销量与溺水事件），需通过差分或协整检验排除
2. **过度拟合**：使用过多参数拟合训练集导致预测失效，建议用 AIC/BIC 准则进行模型选择
3. **概念漂移**：数据生成机制随时间变化（如业务规则调整），需定期重新训练模型
4. **缺失值处理**：不处理缺失值会导致趋势线失真，线性插值和 LOCF（Last Observation Carried Forward）是常用方案
