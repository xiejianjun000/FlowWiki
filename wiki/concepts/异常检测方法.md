# 异常检测方法

## 定义

异常检测方法是通过统计模型、机器学习或规则引擎，从数据中识别偏离正常模式的异常值或异常行为的技术方法。

## 主要类型

- **统计方法**：基于均值/方差的离群点检测
- **规则方法**：基于领域知识的阈值判断
- **机器学习**：无监督聚类 + 有监督分类

## 相关资料

### 统计方法详解

- **3-Sigma 法则**：假设数据正态分布，超出均值 ±3 倍标准差的点为异常。简单但假设强，对非正态数据失效
- **Z-Score 标准化**：将数据归一化后设定阈值（通常 |Z| > 3），适合不同量纲指标的横向比较
- **箱线图法 (IQR)**：基于四分位数间距，Q1 - 1.5×IQR 以下或 Q3 + 1.5×IQR 以上为异常，无需正态假设
- **Grubbs 检验**：假设正态分布，逐一检验最远端样本是否为离群点，适合小样本场景

### 机器学习方法详解

- **Isolation Forest**：通过随机分割隔离异常点，异常点路径短、评分高。计算高效，适合高维数据，无需对正常数据建模
- **DBSCAN 聚类**：基于密度的空间聚类，无法归入任何簇的噪点即为异常。适合任意形状分布，但对密度不均匀数据敏感
- **LOF (局部离群因子)**：通过比较样本与其 k 近邻的局部密度识别异常，适合密度不均匀数据
- **Autoencoder 自编码器**：用正常数据训练重构网络，重构误差大的样本即为异常。适合复杂非线性模式，但需大量正常训练数据

### 规则引擎方法详解

- **静态阈值**：业务专家根据经验设定上下限，简单直观但缺乏自适应能力
- **动态阈值**：基于历史数据的统计特征（如移动平均 ± N×MAD）自动调整阈值，适应周期性波动
- **组合规则**：多条件联动判断，例如"QPS 超过基线 200% 且持续时间超过 5 分钟"才触发告警，有效减少误报

## 方法选择指南

| 场景特征 | 推荐方法 | 原因 |
|---------|---------|------|
| 数据量小、分布已知 | Z-Score / Grubbs | 计算简单，可解释性强 |
| 高维特征、大规模数据 | Isolation Forest | 效率高，无需分布假设 |
| 时间序列周期性明显 | 动态阈值 + STL 残差 | 自适应周期变化 |
| 需要解释异常原因 | 规则引擎 | 每条规则都可追溯业务逻辑 |
| 复杂非线性模式 | Autoencoder | 自动学习正常模式表示 |

## 常用工具

- **PyOD**：统一 API 的 Python 异常检测库，集成 40+ 算法，包含 Isolation Forest、LOF、Autoencoder 等
- **scikit-learn**：提供 EllipticEnvelope、OneClassSVM、Isolation Forest 等经典算法
- **Elasticsearch Machine Learning**：内置于 ELK 栈的无监督异常检测，适合日志/时序场景
- **Grafana Alerting**：基于 Prometheus 数据的可视化告警规则配置和异常通知
